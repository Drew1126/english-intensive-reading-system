### 核心设计原则

| 原则 | 说明 |
|------|------|
| **单服务部署** | 前后端合并为单一 FastAPI 服务，后端通过 `StaticFiles` 托管前端静态文件，无需 CORS |
| **Agent 驱动** | 语法分析、文章获取均通过 Agent 工具链完成，非硬编码 |
| **流式响应** | AI 回答采用 SSE（Server-Sent Events）流式输出，减少等待感 |
| **本地持久化** | 打卡记录、阅读历史以 JSON 文件形式存储于 `data/` 目录，零依赖、易迁移 |

### 4.0 前端静态文件托管（main.py）

FastAPI 通过 `StaticFiles` 将 `frontend/` 目录挂载到根路径，同时支持 HTML 自动解析：

```python
from fastapi.staticfiles import StaticFiles
from pathlib import Path

FRONTEND_DIR = Path(__file__).resolve().parent.parent / "frontend"
app.mount("/", StaticFiles(directory=str(FRONTEND_DIR), html=True), name="frontend")
```

前端 JS 中的 API 请求使用**相对路径** `/api/*`，与页面同源，无需 CORS 配置。

### 4.1 文章获取模块（article_service.py）

**职责：** 通过 Agent 生成当日外刊文章，支持「再来一篇」无限生成，每篇固定 400～450 词。

**实现方式：** 调用 Article Agent，Agent 使用智谱 GLM-4-Flash 生成符合真实外刊风格的文章。文章生成后以 JSON 文件形式存储到 `data/articles/` 目录（文件名格式：`{date}_article_{index}.json`）。

**文章规格（统一标准）：**

| 规格 | 值 |
|------|-----|
| 词数 | **350～500 词（固定范围）** |
| 段落数 | 4 段 |
| 每段句子数 | 5 句 |
| 每句长度 | 18～28 词 |
| 难度对标 | 考研 Part A 标准 |

考研阅读文章主要来源于英美五大主流外刊（The Economist、The Guardian 评论版、The Atlantic、The New Yorker、Scientific American）。

**话题范围（每日自动轮换）：**
Agent 每天自动轮换以下五大话题方向，确保题材多样性：

| 轮换日 | 话题方向 |
|--------|---------|
| Day 1 | 社会伦理 / 文化教育 |
| Day 2 | 经济商业 / 职场趋势 |
| Day 3 | 科技发展 / 人工智能 |
| Day 4 | 生态环境 / 公共健康 |
| Day 5 | 法律政治简版 |

轮换周期为 5 天，基于日期在年中的序数（`tm_yday % 5`）自动选择，保证每天题材不同。

**外刊来源限定：**
- The Economist（经济学人）
- The Guardian 评论版（卫报）
- The Atlantic（大西洋月刊）
- The New Yorker（纽约客）
- Scientific American（科学美国人）

**文风要求：**
- **拒绝简单短句流水账**：避免连续使用小学级别句式
- **不搞学术论文式晦涩**：避免过度嵌套从句、生僻术语堆砌
- **目标文风**：外刊通俗学术文风，有适度长难句但可读性强，类似《经济学人》《卫报》评论版的标准文章
- **句式多样性**：简单句、并列句、复合句交替使用，适当使用倒装、插入语、同位语等考研常见句型

**词汇要求：**
- 以**考研大纲 5500 词**为主体
- 每篇**超纲词严格控制在 8～12 个**，这些超纲词在 `key_vocab` 中标注

**「再来一篇」功能：**
- 用户点击「再来一篇」按钮 → 调用 `POST /api/article/next` → 后端生成一篇新文章
- 新文章自动编号递增（article_0, article_1, article_2, ...），文件名 `{date}_article_{index}.json`
- 所有今日生成的文章均缓存在 `data/articles/` 目录，不会重复生成

**文件缓存策略：**
- 首次请求某日第 N 篇 → 检查 `data/articles/2026-05-02_article_0.json` 是否存在
- 文件不存在 → 调用 Agent 生成 → 将结果序列化为 JSON → 写入文件
- 文件已存在 → 直接 `json.load()` 读取返回，不调用 LLM
- 点击「← 上一篇」可回溯查看今日之前生成过的文章，均从缓存读取

### 4.2 翻译模块（translate_service.py）

**职责：** 对文章进行逐句翻译，翻译结果自动缓存到文章 JSON 文件。

**实现方式：**
- 前端在文章加载完成后**立即自动批量请求翻译**所有段落
- 后端将句子列表批量传入 GLM-4-Flash，要求返回对应中文翻译 JSON 数组
- 翻译结果写回文章对应的 JSON 文件（在 `translations` 字段中追加），下次读取时直接返回已有翻译，无需再次调用 LLM
- 采用 **结构化输出**：在 Prompt 中明确要求模型返回 `{"translations": ["句1译文", "句2译文"]}` 格式的 JSON，后端用 Pydantic 校验

**模型选用：** 翻译任务使用 `glm-4-flash`，速度快、中文质量高，永久免费。

**前端交互：**
- 去除"逐段翻译"按钮，仅保留"显示翻译"开关
- 文章加载后立即自动翻译所有段落，翻译状态实时显示（"加载中..." → "正在翻译..." → 空白）
- 翻译完成后"显示翻译"开关才可操作，避免用户看到空白翻译
- 翻译结果缓存到文件，下次打开同日同难度文章无需重新翻译

### 4.3 打卡模块（checkin_service.py）

**职责：** 记录用户每日打卡、查询打卡历史、计算连续打卡天数。

**实现方式：** 纯文件操作（无需 LLM），所有逻辑在 `checkin_store.py` 中封装：
- 所有打卡记录集中存储在 `data/checkins.json`（一个 JSON 数组）
- 写操作使用 **filelock** 加锁，防止并发请求导致数据丢失
- 查询接口读取文件后在 Python 内存中过滤日期范围，返回指定月份的打卡列表
- 连续打卡天数在读取全部记录后用 Python 排序+遍历逻辑计算，无需 SQL

## 5. Agent 设计详解

### 5.1 Agent 架构选型

采用 **LangChain ReAct Agent + 智谱 GLM-4-Flash**，工具链驱动：

```
用户请求
    │
    ▼
Agent Executor（LangChain）
    │
    ├── 调用 Tool 1：fetch_article_tool     → 生成/获取外刊文章
    ├── 调用 Tool 2：analyze_grammar_tool   → 分析句子语法结构
    ├── 调用 Tool 3：explain_vocabulary_tool → 解释词汇短语
    └── 调用 Tool 4：translate_sentence_tool → 翻译单句
    │
    ▼
流式返回结果（SSE）
```

### 5.3 Reading Agent（阅读辅导）

**文件：** `agents/reading_agent.py`

**功能：** 接受用户选中的句子和问题，分析语法结构、解释词汇、翻译句子。

#### Tool 1：analyze_grammar_tool

**功能：** 深度分析单个英语句子的语法结构

**输入参数：**
```json
{
  "sentence": "原始英文句子",
  "question": "用户的具体问题（可选）",
  "article_context": "文章标题或摘要（用于上下文）"
}
```

**输出格式（GLM-4-Flash 返回）：**
```json
{
  "主干": "主语 + 谓语 + 宾语",
  "句型": "复合句/简单句/复杂句",
  "从句分析": [
    {"类型": "定语从句", "引导词": "which", "修饰成分": "..."}
  ],
  "考研考点": "非限制性定语从句，常见于考研完形填空",
  "解释": "中文逐层分析..."
}
```

**System Prompt 核心内容：**
```
你是考研英语语法专家。分析以下句子时：
1. 先找出句子主干（主谓宾）
2. 标注所有从句类型（定语/状语/名词从句）
3. 识别特殊结构（倒装/虚拟语气/强调句型/分词结构）
```

### 5.4 Agent 的流式调用（SSE 实现）

后端使用 FastAPI 的 `StreamingResponse` + LangChain 的 `astream_events` 实现流式输出：

```python
from fastapi.responses import StreamingResponse
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler

async def stream_agent_response(question: str, sentence: str):
    """生成器函数，yield SSE 格式数据"""
    async for event in executor.astream_events(
        {"input": question, "sentence": sentence},
        version="v2"
    ):
        if event["event"] == "on_chat_model_stream":
            chunk = event["data"]["chunk"].content
            if chunk:
                yield f"data: {json.dumps({'text': chunk})}\n\n"
    yield "data: [DONE]\n\n"

@router.get("/api/agent/ask")
async def ask_agent(question: str, sentence: str):
    return StreamingResponse(
        stream_agent_response(question, sentence),
        media_type="text/event-stream"
    )
```

前端用 `EventSource` 接收（使用相对路径）：
```javascript
const source = new EventSource(`/api/agent/ask?question=${q}&sentence=${s}`);
source.onmessage = (e) => {
  if (e.data === '[DONE]') { source.close(); return; }
  const { text } = JSON.parse(e.data);
  appendToPanel(text);  // 逐字追加到面板
};
```

### 6.2 文章渲染模块（article.js）

**实现方式：**
- 后端返回 JSON 格式的文章（段落数组 → 句子数组）
- 前端遍历句子数组，将每个句子包裹在 `<span class="sentence" data-idx="n">` 中
- 绑定 `click` 事件：点击句子 → 高亮 + 触发 Agent 问答面板更新
- **重点词汇高亮**：段落中的重点词汇（`key_vocab`）用不同颜色高亮标注，不再显示在段落下方的词汇表
- **词汇释义弹窗**：点击高亮词汇 → 显示浮动 tooltip 展示词义，点击页面其他地方自动关闭
- 翻译开关：`toggle` 状态变量控制每个句子下方的 `<div class="translation">` 的 `display` 属性
- 选中状态通过 CSS class 管理（`.sentence.selected` 样式高亮）
- **翻译预加载**：文章渲染完成后立即调用 `preloadAllTranslations()`，逐段请求翻译 API，翻译结果写回文章 JSON 文件缓存

### 6.3 逐句翻译模块

**实现方式（批量预加载 + 文件缓存）：**

文章加载完成后立即自动翻译所有段落，翻译结果缓存到文章 JSON 文件：
1. 前端检查文章各段 `translations` 字段是否已填满
2. 已全部填满 → 直接渲染翻译，无需调用 API
3. 未填满 → 逐段调用 `POST /api/translate`，后端将翻译结果写回文件
4. 翻译过程中显示"正在翻译..."状态，完成后"显示翻译"开关才可操作
5. 去除"逐段翻译"按钮，仅保留"显示翻译"开关一个入口

### 6.4 文章导航（「再来一篇」）

**实现方式：**
- 去除难度选择器，Header 改为文章导航控件：「← 上一篇」+「第 N 篇」+「再来一篇」
- 首次加载：`GET /api/article/daily?article_index=0`，获取当日第一篇文章（如不存在则自动生成）
- 点击「再来一篇」：`POST /api/article/next`，后端生成新文章（编号自动递增），前端立即渲染
- 点击「← 上一篇」：`GET /api/article/daily?article_index=N-1`，从缓存读取前一篇，无需重新生成
- 所有当日生成的文章缓存在 `data/articles/` 目录（`{date}_article_0.json`、`{date}_article_1.json`...）
- 文章状态通过 `article_index` 追踪，Header 实时显示"第 N 篇"

### 核心工具类设计（`storage/file_store.py`）

文件读写统一通过该工具类操作，封装以下能力：

| 方法 | 功能 |
|------|------|
| `read_json(path)` | 读取 JSON 文件，文件不存在返回默认值（`None` 或 `[]`） |
| `write_json(path, data)` | 加文件锁后序列化写入，`indent=2` 保持可读性 |
| `append_to_list(path, item)` | 加锁读取 → append → 写回，用于打卡和问答追加 |
| `ensure_dir(path)` | 确保目录存在（`pathlib.Path.mkdir(parents=True, exist_ok=True)`） |

**文件锁使用示例（防并发损坏）：**

```python
from filelock import FileLock
import json, pathlib

def write_json(path: str, data) -> None:
    lock_path = path + ".lock"
    with FileLock(lock_path, timeout=5):
        pathlib.Path(path).parent.mkdir(parents=True, exist_ok=True)
        with open(path, "w", encoding="utf-8") as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
```

### ☁️ 云服务器部署推荐（国内 BGP 线路）

**部署步骤：**

1.  **上传代码**：使用 SFTP 工具（如 FileZilla）将 `backend` 和 `frontend` 上传到服务器 `/home/ubuntu/english-app/`。
2.  **配置环境**：
    ```bash
    sudo apt update
    sudo apt install python3 python3-pip python3-venv -y
    cd /home/ubuntu/english-app/backend
    pip3 install -r requirements.txt
    ```
3.  **填写 Key**：在服务器创建 `.env` 文件并填入你的 `ZHIPU_API_KEY`。
4.  **启动服务**：
    ```bash
    # 使用 nohup 后台运行
    nohup python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &
    ```
5.  **访问**：浏览器打开 `http://<你的服务器公网IP>:8000`。

### 生产部署（单机）

| 组件 | 方案 |
|------|------|
| 后端运行 | **Uvicorn** + **Gunicorn**（多 worker） |
| 反向代理 | **Nginx**（代理 /api/* 和静态文件） |
| 进程管理 | **systemd** 或 **PM2** |
| 数据存储 | `data/` 目录 JSON 文件，建议定期备份该目录 |
| HTTPS | **Let's Encrypt** + Certbot |

### Nginx 配置要点

```nginx
server {
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_buffering off;          # SSE 必须关闭缓冲
        proxy_cache off;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    location /static/ {
        alias /var/www/app/frontend/;  # 或直接由 FastAPI 托管
    }
}
```
